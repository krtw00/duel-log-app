# モバイル対応設計

スマートフォンでの閲覧・入力をサポート。

> **❌ 未実装:** 計画中機能

---

## 現状分析

### 対応済み

| コンポーネント | 対応内容 |
|--------------|---------|
| AppBar | モバイル用ハンバーガーメニュー |
| NavigationDrawer | `temporary`モード |
| StatsDisplayCards | レスポンシブグリッド |
| DuelFormDialog | xs時`fullscreen` |
| DuelTable | モバイル用カード表示 |
| チャート | レスポンシブ高さ |

### 要改善

| 優先度 | 問題 |
|-------|------|
| 🔴 Critical | 相性表のモバイル対応なし |
| 🟠 High | ラジオボタンが横並び固定 |
| 🟠 High | padding `pa-6`が大きい |
| 🟡 Medium | ダイアログmax-width 500px |
| 🟡 Medium | タッチターゲット < 44px |
| 🟢 Low | ボトムナビゲーションなし |

---

## ブレークポイント

| 名前 | 幅 | デバイス |
|-----|---|---------|
| xs | < 600px | スマートフォン（縦） |
| sm | 600-959px | スマートフォン（横）、小型タブレット |
| md | 960-1279px | タブレット |
| lg | 1280-1919px | デスクトップ |
| xl | >= 1920px | 大画面 |

---

## 設計原則

| 原則 | 説明 |
|------|------|
| モバイルファースト | xs → lg の順でスタイル適用 |
| タッチ対応 | 最小タッチターゲット 44x44px |
| 縦スクロール優先 | 横スクロールは極力避ける |
| コンテンツ優先 | 装飾より情報を優先 |

---

## 改善計画

### Phase 1: データテーブル最適化（Critical）

| 項目 | 対応 |
|------|------|
| DuelTable | ✅ モバイルカード表示実装済み |
| 相性表 | アコーディオン形式に変更 |

### Phase 2: 入力フォーム最適化（High）

| 項目 | 対応 |
|------|------|
| ラジオボタン | モバイルで縦並び |
| padding | xsで`pa-2`〜`pa-3` |

### Phase 3: ナビゲーション改善（Low）

| 項目 | 対応 |
|------|------|
| ボトムナビゲーション | 主要機能へのアクセス |
| スワイプジェスチャー | ページ遷移 |

---

## 実装例: レスポンシブ判定

```typescript
import { useDisplay } from 'vuetify'
const { xs, sm } = useDisplay()
const isMobile = computed(() => xs.value || sm.value)
```
