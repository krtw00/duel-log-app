# 開発ガイドライン

このディレクトリは、開発を始めるために必要な情報、環境構築、コーディング規約に関するドキュメントをまとめたものです。

## 📚 ドキュメント一覧

- **[development-guide.md](./development-guide.md)** - ブランチ戦略、コミット規約（Conventional Commits）、Pull Request ワークフロー、開発環境のセットアップについて説明します。
- **[environment-setup.md](./environment-setup.md)** - バックエンド・フロントエンド・Docker環境のセットアップ、環境変数設定、依存ツールのインストール方法について説明します。
- **[development-tutorial.md](./development-tutorial.md)** - 初心者向けの開発環境構築チュートリアルと便利なコマンド集。

## 🎯 このセクションを読むべき人

- **新規開発者**: 環境構築と開発フローを理解したい場合（**最初にこちらを読んでください**）
- **既存開発者**: ブランチ戦略や規約を再確認したい場合
- **CI/CD担当者**: コミット規約やPRルールを確認したい場合

## 📖 読む順序

**初めての方:**
1. [environment-setup.md](./environment-setup.md) - 開発環境を構築
2. [development-guide.md](./development-guide.md) - ブランチ戦略とワークフローを理解
3. [development-tutorial.md](./development-tutorial.md) - よく使うコマンドを確認

**既存開発者:**
- [development-guide.md](./development-guide.md) - コミット規約やブランチ戦略を確認
